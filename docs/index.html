Features

Everything you need to
segment tennis matches

Pose Detection

Uses YOLOv8 pose estimation to track player movements and keypoints throughout the match.

LSTM Analysis

Deep learning model analyzes temporal patterns to identify when points are being played.

Smart Segmentation

Automatically cuts out dead time between points, keeping only the action.

CSV Export

Optionally export point timestamps to CSV for further analysis or visualization.

GUI & CLI

Use the browser-based interface for simplicity, or the CLI for scripting and automation.

100% Local

All processing happens on your machine. No data leaves your computer.

How It Works

The processing pipeline

01

Pose Extraction

YOLOv8 detects and tracks player poses frame by frame

02

Feature Engineering

Extract motion features and normalize keypoint positions

03

LSTM Inference

Neural network predicts play/no-play probability per frame

04

Video Output

Segments combined into final video with only rallies

Installation

Get up and running
in minutes

Requirements

  • Python 3.10+
  • 8GB+ RAM recommended
  • GPU optional (MPS/CUDA supported)
bash 
# Clone and install
git clone https://github.com/iroblesrazzaq/RallyVision.git
cd RallyVision
pip install .

Models are included in the repo. YOLO weights auto-download on first run.

Usage

Two ways to use RallyVision

Browser-Based Interface

Launch the GUI with a single command. Drag and drop your video, adjust settings if needed, and download the result.

rallyvision gui
  • Drag & drop video upload
  • Real-time progress tracking
  • Advanced settings panel
  • Download results directly

Command Line Interface

Perfect for scripting, batch processing, or when you prefer the terminal.

bash 
# Basic usage
rallyvision --video "match.mp4"

# With CSV output
rallyvision --video "match.mp4" --write-csv

# Custom output directory
rallyvision --video "match.mp4" --output-dir "./processed"

# Use config file
rallyvision --config config.toml
